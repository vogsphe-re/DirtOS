This is the standard DirtOS operating loop.

## 1. Plan structure

- Create or update [Environments](../reference/environments-locations.md)
- Add [Locations](../reference/environments-locations.md#location-fields)
  for beds, tents, and trays

## 2. Add plants

- Confirm species metadata in [Plants and Species](../reference/plants-species.md)
- Create individual plants and place them in locations
- Assign labels and notes for identification

## 3. Define maintenance

- Create [Schedules](../reference/schedules.md) for water, feed, and maintenance
- Verify recurrence logic and next run behavior

## 4. Monitor conditions

- Add [Sensors](../reference/sensors.md)
- Set [Sensor Limits](../reference/sensors.md#limit-fields)
- Review readings and outlier trends

## 5. Record outcomes

- Track issues in [Issues](../reference/issues.md)
- Add timeline entries in [Journal](../reference/journal.md)
- Record yields in [Reports](../reference/reports.md)

## 6. Protect data

- Export backups on a schedule
- Validate import behavior periodically
